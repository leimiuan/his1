# 智慧HIS系统操作手册

版本：v1.0  
适用目录：D:\C++\his1-main\his1-main\xitong  
适用系统：Windows 控制台版

## 1. 系统简介

本系统为命令行版医院信息系统（HIS），涵盖患者、医生、管理员、120急救调度四大业务终端，支持门急诊分流、住院与ICU流转、手术室状态管理、财务与库存管理、历史记录追踪等功能。

## 2. 启动前准备

### 2.1 环境要求

- Windows 系统
- 已安装 GCC（Makefile 默认使用 `gcc`）
- 控制台支持 UTF-8（程序启动后会自动设置）

### 2.2 编译与运行

在 xitong 目录执行：

```bash
make
.\his_system.exe
```

说明：

- 可执行文件名为 `his_system.exe`
- 首次运行会从 `data` 目录加载基础数据

### 2.3 退出与数据保存

从主菜单选择 `0` 安全关闭系统，程序会自动将内存数据回写到 `data` 目录文本文件。

## 3. 数据文件说明（data目录）

- `departments.txt`：科室信息
- `doctors.txt`：医生信息
- `outpatients.txt`：门诊患者
- `inpatients.txt`：住院患者
- `records.txt`：诊疗记录与账单
- `beds.txt`：床位信息
- `medicines.txt`：药品与库存
- `ambulances.txt`：救护车实时状态
- `ambulance_logs.txt`：120出勤历史
- `operating_rooms.txt`：手术室状态（空闲/手术中/消毒）
- `schedules.txt`：医生排班信息

## 4. 主菜单导航

程序主界面提供 4 个业务入口：

- `1` 患者服务大厅
- `2` 医护人员工作站
- `3` 决策支持中心（管理员）
- `4` 120急救调度中心
- `0` 安全关闭系统

## 5. 患者端操作（入口1）

登录后可使用：

- 个人资料与住院/ICU状态查询
- 预约挂号（含限号规则与智能推荐）
- 实时排队查询
- 待缴费账单结算
- 余额充值
- 呼叫120救护车
- 医生值班表查询
- 历史诊疗记录查询
- 财务报表与电子发票

推荐流程：

1. 先充值（若余额不足）
2. 挂号（普通门诊/急诊）
3. 查看排队
4. 就诊后缴费
5. 在财务报表页面生成电子发票

## 6. 医生端操作（入口2）

医生需输入工号登录，系统会根据当日排班自动识别门诊/急诊模式。

常用菜单：

- 候诊队列查看与呼叫下一位
- 结束当前诊疗任务
- 开具处方 / 药房核销
- 开具体检单 / 生成体检报告
- 手术预约与计费
- 患者入院收治
- ICU流转
- 本科室住院名单
- 手术室实时追踪
- 个人工作量统计

注意：

- 手术预约有并发防撞锁，同一患者、同一主刀医生不可同时占用两台手术
- 如提示“患者正在某手术室接受手术”，需先在管理员端执行“结束手术”流程

## 7. 管理员端操作（入口3）

### 7.1 登录

管理员密码：`admin123`

### 7.2 核心能力

- 医生注册/删除
- 床位利用率与调配
- 药品入库与库存审计
- 财务报表与就诊流量分析
- 诊疗记录增删改查（含文件批量导入）
- 排班表管理
- 手术室排班与调度管理
- 全院候诊队列看板
- 病房流转控制

### 7.3 手术室状态流转（重点）

进入：管理员菜单 `18 手术室排班与调度管理`

手术室状态建议按以下顺序流转：

1. `2 安排手术`（空闲 -> 手术中）
2. `3 结束手术并进入消毒维护`（手术中 -> 消毒中）
3. `4 消毒完成释放手术室`（消毒中 -> 空闲）

## 8. 120急救调度中心（入口4）

支持功能：

- 救护车实时状态看板
- 派单出车（自动写入出勤历史）
- 回场与消毒登记
- 出勤历史查询
- 联动查看急诊手术室状态

场景说明：

- 回场时可选择将危重患者直接推入急诊绿色通道，系统会自动生成急诊记录并置顶提醒医生。

## 9. 常见问题（FAQ）

### 9.1 为什么“明明有空手术室”却不能给该患者做手术？

因为系统启用了并发防撞锁：同一患者在同一时刻只能处于一个手术室的“手术中”状态。

### 9.2 如何把患者从手术室里“移出来”？

在管理员端执行：

1. `18` 进入手术室调度中心
2. 选择 `3 结束手术并进入消毒维护`，输入手术室编号（如 OR-03）
3. 再选择 `4 消毒完成释放手术室`

执行第2步后，患者会从该手术室记录中清空；执行第3步后手术室恢复可预约。

### 9.3 程序数据什么时候落盘？

选择主菜单 `0` 安全退出时，系统会自动保存到 `data` 目录。

## 10. 运维建议

- 每次培训或演示前，先备份 `data` 目录
- 演示手术流程时，建议先查看手术室看板确认状态
- 发生异常退出后，优先检查 `records.txt`、`operating_rooms.txt`、`beds.txt`
