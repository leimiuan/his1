# HIS系统操作手册

版本：2026-04-01  
适用项目：`D:\C++\his1-main\his1-main\xitong`  
适用程序：`his_system.exe`

## 1. 系统简介

本系统为控制台版医院信息系统（HIS），当前包含 4 个业务入口：

- 患者端
- 医生端
- 管理端
- 急救中心

系统支持以下核心业务：

- 患者建档、充值、挂号、缴费、排队查询
- 医生叫号、接诊、开处方、发药、开检查单、生成检查报告
- 住院收治、押金校验、床位分配、ICU 转入
- 手术室使用与释放
- 管理端记录维护、药品入库、财务与床位监控
- 救护车调度、回场登记、急诊联动

## 2. 启动说明

### 2.1 进入目录

项目目录：`D:\C++\his1-main\his1-main\xitong`

### 2.2 编译命令

建议在 PowerShell 中执行：

```powershell
New-Item -ItemType Directory -Force tmpbuild | Out-Null
$env:TMP='D:\C++\his1-main\his1-main\xitong\tmpbuild'
$env:TEMP='D:\C++\his1-main\his1-main\xitong\tmpbuild'
mingw32-make clean all
```

### 2.3 运行命令

```powershell
.\his_system.exe
```

### 2.4 退出与保存

在主菜单输入 `0`，系统会自动保存数据并安全退出。

## 3. 主菜单说明

程序启动后会显示以下入口：

- `1` 患者端
- `2` 医生端
- `3` 管理端
- `4` 急救中心
- `0` 保存并退出

## 4. 患者端操作

### 4.1 入口

主菜单输入 `1` 后进入患者端。

患者端首页功能：

- `1` 登录
- `2` 新建患者档案
- `0` 返回

### 4.2 新建患者档案

适用场景：系统中还没有该患者。

操作步骤：

1. 进入患者端。
2. 选择 `2 新建患者档案`。
3. 依次输入姓名、年龄、性别、医保类型。
4. 系统自动生成患者编号。

说明：后续登录、挂号、缴费都需要使用该患者编号。

### 4.3 患者登录后功能

登录成功后可使用以下菜单：

- `1` 个人信息
- `2` 挂号
- `3` 排队状态
- `4` 缴费
- `5` 充值
- `6` 呼叫 120 救护车
- `7` 查看值班表
- `8` 查看病历
- `9` 费用统计
- `0` 返回

### 4.4 充值

操作步骤：

1. 登录患者账号。
2. 选择 `5 充值`。
3. 输入充值金额。
4. 系统提示充值成功并显示新余额。

### 4.5 挂号

系统支持两类挂号：

- 门诊挂号：5.00 元
- 急诊挂号：50.00 元

操作步骤：

1. 登录患者账号。
2. 如余额不足，先执行充值。
3. 选择 `2 挂号`。
4. 选择挂号类型。
5. 输入目标医生工号。
6. 系统生成挂号记录，等待缴费。
7. 再进入 `4 缴费` 完成支付。

挂号规则：

- 全院每日挂号总量有限制。
- 单个患者每日挂号次数有限制。
- 同一患者同一天同一科室只允许挂号一次。
- 单个医生当日号源满额后，系统会提示同科室可选医生。

### 4.6 缴费

操作步骤：

1. 登录患者账号。
2. 选择 `4 缴费`。
3. 查看待缴费记录列表。
4. 输入要支付的记录编号。
5. 支付成功后，系统会生成电子票据。

说明：挂号、处方、检查单等记录通常都需要先缴费，再进入后续流程。

### 4.7 排队状态

功能说明：

- 查看当前候诊记录
- 显示队列号
- 显示前方人数
- 显示预计等待时间

如果挂号未缴费，系统会提示“有挂号记录尚未缴费”。

### 4.8 查看病历与费用统计

- `8 查看病历`：查看当前患者的全部记录
- `9 费用统计`：查看累计个人支付金额

### 4.9 呼叫 120

操作步骤：

1. 登录患者账号。
2. 选择 `6 呼叫 120 救护车`。
3. 系统自动寻找空闲救护车并完成派车。

## 5. 医生端操作

### 5.1 入口

主菜单输入 `2` 后进入医生端。

登录方式：输入医生工号。

说明：

- 系统会根据当天排班自动识别当前为门诊模式还是急诊模式。
- 如果当天无排班记录，系统默认以门诊模式进入。

### 5.2 医生端菜单

医生端主要功能如下：

- `1` 查看候诊队列
- `2` 呼叫下一位患者
- `3` 结束当前接诊
- `4` 开立处方
- `5` 药房发药
- `6` 开立检查单
- `7` 生成检查报告
- `8` 安排手术
- `9` 办理住院收治
- `10` 转入住院患者到 ICU
- `11` 查看科室住院患者
- `12` 查看手术室看板
- `13` 查看救护调度记录
- `14` 查看统计信息
- `0` 返回

### 5.3 标准接诊流程

推荐按以下顺序操作：

1. `1 查看候诊队列`
2. `2 呼叫下一位患者`
3. 对当前患者执行处方、检查、手术或住院等操作
4. 必要时通知患者去缴费
5. `3 结束当前接诊`

说明：

- 未先叫号时，不能直接对患者开处方或开检查单。
- 当前已有接诊患者时，不能再次叫号。

### 5.4 开立处方

操作步骤：

1. 先呼叫一位患者。
2. 选择 `4 开立处方`。
3. 选择处方类型：普通、急诊、慢病。
4. 输入药品编号与数量。
5. 系统生成处方记录。

处方有效期：

- 普通处方：7 天
- 急诊处方：3 天
- 慢病处方：30 天

### 5.5 药房发药

操作步骤：

1. 选择 `5 药房发药`。
2. 输入处方记录编号。
3. 系统校验：
   - 处方是否存在
   - 是否已缴费
   - 是否已发药
   - 是否过期
   - 库存是否足够
4. 校验通过后完成扣减库存并发药。

### 5.6 开立检查单与生成检查报告

开检查单：

1. 先呼叫患者。
2. 选择 `6 开立检查单`。
3. 输入检查项目和费用。
4. 系统生成待缴费检查记录。

生成检查报告：

1. 选择 `7 生成检查报告`。
2. 输入检查记录编号。
3. 检查记录必须已经缴费。
4. 输入检查结果摘要。
5. 系统生成检查报告。

### 5.7 办理住院收治

操作步骤：

1. 先呼叫一位门诊患者。
2. 选择 `9 办理住院收治`。
3. 输入预计住院天数。
4. 系统计算最低押金：`200 x 住院天数`。
5. 输入实收押金。
6. 选择病房类型：
   - `1` ICU
   - `2` 普通
   - `3` 陪护
7. 输入预计出院日期。
8. 系统自动分配可用床位并完成转入住院。

收治规则：

- 押金必须不低于最低押金。
- 押金必须是 100 的整数倍。
- 合法收治时，系统会优先匹配本科室对应病房类型的空床。

### 5.8 ICU 转入

操作步骤：

1. 选择 `10 转入住院患者到 ICU`。
2. 输入住院患者编号。
3. 系统检查本科室 ICU 床位。
4. 若有空床，则释放原床位并转入 ICU。
5. 若 ICU 已满，系统触发告警提示。

### 5.9 安排手术

操作步骤：

1. 先呼叫患者。
2. 选择 `8 安排手术`。
3. 输入手术名称和预计费用。
4. 系统自动寻找空闲手术室。
5. 安排成功后，手术室状态变为“使用中”。

限制规则：

- 同一医生不能同时占用多个正在使用中的手术室。
- 同一患者不能同时出现在多个手术室中。
- 若无空闲手术室，系统会提示无法安排。

### 5.10 医生统计

选择 `14 查看统计信息` 可以查看：

- 今日接诊量
- 累计收入

说明：今日接诊量基于“当日已缴费且已叫号/已完成”的挂号记录统计。

## 6. 管理端操作

### 6.1 登录

主菜单输入 `3` 后进入管理端。

管理员密码：`admin123`

### 6.2 管理端菜单

- `1` 新增记录
- `2` 从文件导入记录
- `3` 修改记录（原记录作废）
- `4` 作废一条记录
- `5` 按科室查询记录
- `6` 按医生查询记录
- `7` 按患者查询记录
- `8` 按日期范围查询
- `9` 新增医生
- `10` 删除医生
- `11` 药品入库
- `12` 查看库存报表
- `13` 查看财务报表
- `14` 查看医生工作量趋势
- `15` 查看床位分配监控
- `16` 查看全院排队监控
- `17` 手术室管理
- `0` 返回

### 6.3 新增医生

操作步骤：

1. 进入管理端。
2. 选择 `9 新增医生`。
3. 输入医生工号、姓名、职级、科室编号、值班时间。
4. 保存后即可在挂号和医生登录中使用。

### 6.4 药品入库与库存查看

- `11 药品入库`：输入药品编号和新增数量
- `12 查看库存报表`：查看全部药品库存和预警状态

### 6.5 财务与运营监控

- `13 查看财务报表`：查看诊疗、检查、住院、手术、检查单等收入汇总
- `14 查看医生工作量趋势`：按医生展示当日工作量
- `15 查看床位分配监控`：查看各科室床位使用率，超过阈值时显示告警
- `16 查看全院排队监控`：查看全院急诊与门诊候诊情况

### 6.6 手术室管理

进入方式：管理端选择 `17 手术室管理`

功能菜单：

- `1` 查看手术室看板
- `2` 结束手术并转入消杀
- `3` 完成消杀并释放手术室
- `0` 返回

手术室状态流转：

1. 医生安排手术后，手术室进入“使用中”
2. 管理端执行 `2 结束手术并转入消杀`
3. 患者和医生会从该手术室记录中清空
4. 手术室进入“消杀中”
5. 管理端执行 `3 完成消杀并释放手术室`
6. 手术室恢复“空闲”

这部分是手术室患者移出流程的标准操作入口。

## 7. 急救中心操作

### 7.1 入口

主菜单输入 `4` 后进入急救中心。

### 7.2 功能菜单

- `1` 查看全院救护车实时监控
- `2` 紧急派单出车
- `3` 车辆回场及消杀登记
- `4` 查看救护车历史出勤表
- `5` 联动查看急诊手术室实时状态
- `0` 返回

### 7.3 紧急派车

操作步骤：

1. 进入急救中心。
2. 选择 `2 紧急派单出车`。
3. 输入空闲车辆编号。
4. 输入急救现场地址或定位。
5. 系统将车辆状态改为“执行任务”，并自动写入出勤记录。

### 7.4 车辆回场与急诊联动

操作步骤：

1. 选择 `3 车辆回场及消杀登记`。
2. 输入回场车辆编号。
3. 系统将车辆状态恢复为待命。
4. 如车上有危重患者，可选择进入“急诊绿色通道”。
5. 输入患者编号和负责接诊的急诊医生工号。
6. 系统自动生成急诊记录并推送到医生候诊队列。

### 7.5 出勤记录与手术室联动

- `4 查看救护车历史出勤表`：查看所有历史派车记录
- `5 联动查看急诊手术室实时状态`：用于调度人员提前判断手术室空闲情况

## 8. 典型测试流程建议

### 8.1 门诊接诊流程

1. 患者新建档案
2. 患者充值
3. 患者挂号并缴费
4. 医生登录并查看队列
5. 医生呼叫患者
6. 医生开处方或检查单
7. 患者再去缴费
8. 医生结束接诊

### 8.2 检查单闭环流程

1. 医生呼叫患者
2. 医生开检查单
3. 患者缴费
4. 医生生成检查报告
5. 患者在病历中查看记录

### 8.3 住院收治流程

1. 患者完成门诊挂号并缴费
2. 医生叫号
3. 医生办理住院收治
4. 输入合法押金
5. 系统分配床位
6. 管理端或医生端查看住院信息

### 8.4 ICU 熔断流程

1. 先占满某科室 ICU 床位
2. 医生执行 ICU 转入
3. 系统提示 ICU 告警
4. 若 ICU 有空床，则应可正常转入

### 8.5 手术室流程

1. 医生安排手术
2. 管理端进入手术室管理
3. 执行“结束手术并转入消杀”
4. 确认患者已从手术室记录中移出
5. 执行“完成消杀并释放手术室”
6. 确认手术室重新变为空闲

## 9. 常见问题

### 9.1 为什么挂号后看不到排队？

通常是因为挂号记录还未缴费。请先到患者端执行缴费。

### 9.2 为什么医生端不能直接开处方？

必须先通过 `呼叫下一位患者` 选中当前接诊患者后，才能进行处方、检查、手术、收治等操作。

### 9.3 为什么住院收治失败？

重点检查以下几项：

- 押金是否达到最低标准
- 押金是否为 100 的整数倍
- 对应科室和病房类型是否还有可用床位
- 当前患者是否已经是住院状态

### 9.4 如何把患者从手术室里移出？

请进入管理端：

1. `17 手术室管理`
2. `2 结束手术并转入消杀`
3. 输入手术室编号

执行后，患者会从当前手术室记录中清空。

如果还要让手术室重新可预约，再执行：

1. `17 手术室管理`
2. `3 完成消杀并释放手术室`

### 9.5 系统数据什么时候保存？

从主菜单输入 `0` 保存并退出` 时，系统会自动写回 `data` 目录中的数据文件。

## 10. 数据文件说明

系统主要数据文件位于 `data` 目录：

- `departments.txt`：科室信息
- `doctors.txt`：医生信息
- `outpatients.txt`：门诊患者
- `inpatients.txt`：住院患者
- `records.txt`：业务记录与账单
- `beds.txt`：床位信息
- `medicines.txt`：药品库存
- `ambulances.txt`：救护车信息
- `ambulance_logs.txt`：救护车出勤记录
- `operating_rooms.txt`：手术室状态
- `schedules.txt`：医生排班

## 11. 使用建议

- 做功能测试前，建议先备份 `data` 目录。
- 测试住院、ICU、手术室时，建议同步关注 `床位分配监控` 和 `手术室看板`。
- 如果程序异常退出，优先检查 `records.txt`、`beds.txt`、`operating_rooms.txt`。
