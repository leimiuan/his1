# HIS系统手把手操作手册（全端版）

版本：2026-04-02
适用目录：`D:\C++\his1-main\xitong`
程序：`his_system.exe`
编码规范：全部使用 UTF-8

---

## 1. 先启动系统（第一次用）

1. 打开 PowerShell，进入项目目录：`D:\C++\his1-main\xitong`
2. 执行编译：

```powershell
$env:TMP='D:\C++\his1-main\xitong\tmp'
$env:TEMP='D:\C++\his1-main\xitong\tmp'
mingw32-make clean all
```

3. 运行程序：

```powershell
.\his_system.exe
```

4. 主菜单会显示：
- `1` 患者端
- `2` 医生端
- `3` 管理端
- `4` 急救中心
- `0` 保存并退出

---

## 2. 新手先记这 4 条

1. 每输入一项都要按一次回车。
2. 看到“按回车继续”时，直接按回车。
3. 任何端口想返回上一层，输入 `0`。
4. 退出系统请走主菜单 `0`，会自动保存数据。

---

## 3. 一条完整流程（照着按就能通）

目标：新建患者 -> 挂号 -> 缴费 -> 医生接诊 -> 开处方 -> 缴处方费 -> 药房发药。

### 第一步：新建患者
1. 主菜单输入 `1`（患者端）
2. 输入 `2`（挂号新患者）
3. 按提示输入姓名、年龄、性别、医保类型
4. 记下系统生成的患者编号（例如 `P1775...`）

### 第二步：患者充值
1. 患者端输入 `1`（登录）
2. 输入患者编号
3. 患者主页输入 `5`（充值）
4. 输入金额，例如 `500`

预期：余额上升。

### 第三步：挂号
1. 患者主页输入 `2`（挂号）
2. 选 `1` 门诊挂号（5.00）或 `2` 急诊挂号（50.00）
3. 输入医生工号（建议 `D001`）

### 第四步：缴挂号费
1. 患者主页输入 `4`（缴费）
2. 输入挂号记录编号

预期：
- 记录状态变已缴费
- 患者余额减少

### 第五步：医生叫号并开处方
1. 回主菜单输入 `2`（医生端）
2. 输入 `D001`
3. 输入 `2`（呼叫下一位患者）
4. 输入 `4`（开立处方）
5. 处方类型选 `1`（普通）
6. 药品编号输入 `M100`
7. 数量输入 `1`

### 第六步：患者缴处方费
1. 回患者端登录同一患者
2. 输入 `4`（缴费）
3. 输入处方记录编号

### 第七步：医生药房发药
1. 回医生端
2. 输入 `5`（药房发药）
3. 输入处方记录编号

预期：
- 提示“发药成功”
- 处方记录追加 `|DISPENSED`
- 药品库存减少

## 8. 你重点关心的“数值变化”验收点

1. 充值后：余额增加。
2. 缴费后：余额减少，记录状态变已缴费。
3. 发药后：库存减少，记录带 `|DISPENSED`。
4. 新增医生后：医生数量 +1；删除后回退。
5. 药品进库后：库存增加。
6. 住院收治后：住院患者数/占床数不下降。
7. 急救派车+回场后：出勤日志增加。
8. 绿色通道触发后：新增急诊记录。

## 9. 非法输入怎么测（建议每次改动后跑）

1. 充值输入负数：应提示金额无效。
2. 挂号输入不存在医生：应提示未找到医生。
3. 缴费输入错误记录号：应提示未找到账单。
4. 药房发药输入错误记录号：应提示未找到处方记录。
5. 发药时处方未缴费：应拦截。
6. 发药时库存不足：应拦截。
7. 管理员输错密码：应拦截。
8. 救护车派单输入不存在车辆：应拦截。

## 10. 常见问题

### Q1：挂号后缴费没扣费
检查：记录号是否正确、是否待缴费、余额是否足够。

### Q2：处方后找不到药房记录号
检查：处方是否创建成功、患者是否缴了该处方、输入是否同一记录号。

### Q3：进不了药房发药入口
标准路径：主菜单 `2` -> 医生工号（如 D001）-> 选 `5`。
